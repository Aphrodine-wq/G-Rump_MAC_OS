# Documentation Skill

When creating documentation:

1. Start with the reader's goal (what they need to do)
2. Use simple language and short sentences
3. Include examples and code snippets where helpful
4. Keep README, API docs, and comments in sync
5. Structure with headers and lists for scanning
6. Update docs when behavior changes

Prefer clarity over completeness.
