# Debugging Skill

When debugging:

1. Reproduce the issue reliably first
2. Gather error messages, stack traces, and logs
3. Form hypotheses and test them one at a time
4. Use divide-and-conquer (binary search, breakpoints)
5. Check recent changes and dependencies
6. Propose minimal, targeted fixes with tests

Document the root cause and fix before applying changes.
