# Refactoring Skill

When refactoring:

1. Understand existing behavior before changing
2. Make small, incremental changes with clear intent
3. Preserve or add tests to lock in behavior
4. Improve names, extract functions, reduce duplication
5. Document the refactoring approach before starting
6. Avoid mixing refactors with new features

Prefer readability and maintainability over cleverness.
