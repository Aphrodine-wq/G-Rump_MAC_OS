# Code Review Skill

When reviewing code:

1. Check for correctness and edge cases
2. Suggest security improvements (injection, XSS, auth)
3. Recommend style and consistency fixes
4. Identify performance bottlenecks
5. Note missing tests or documentation
6. Propose clearer naming or structure

Output findings as a structured review with severity (critical, major, minor).
