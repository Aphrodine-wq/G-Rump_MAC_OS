# Testing Skill

When working with tests:

1. Write tests for new features before or with the code
2. Prefer clear, isolated unit tests over brittle integration tests
3. Use descriptive test names that explain the scenario
4. Mock external dependencies to keep tests fast
5. Add regression tests for bugs that are fixed
6. Run tests locally before suggesting changes

Suggest test coverage for critical paths and edge cases.
